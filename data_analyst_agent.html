Data Analyst Agent

1. Problem Definition / Objective

The objective of this solution is to enable business users to perform complex data analysis through natural language queries — without writing SQL — and to receive both tabular and visual results instantly.

2. Data Available

The system connects directly to existing company databases (e.g., transactional, CRM, or financial data sources). It can access multiple tables and schemas, using metadata information to understand field names, data types, and relationships between tables.

3. Solution Deployment

A Data Analytics Agent powered by OpenAI acts as the core of the system.

  • Users interact through a Streamlit interface, submitting questions in natural language.
  • The agent interprets the question, generates the corresponding SQL query, validates it against the schema, and executes it on the connected database.
  • The query results are then transformed into a data table and an interactive visualization (chart or graph), which are displayed back to the user through Streamlit.

4. Tools and Technologies

  • OpenAI API: Natural language understanding, SQL generation, and validation.
  • Streamlit: Web-based user interface for input, visualization, and interactivity.
  • SQL Databases: Primary data sources for querying and analytics.
  • Python: Backend logic and integration layer connecting Streamlit, OpenAI, and the databases.
  • Visualization Libraries: (e.g., Plotly, Matplotlib) for generating dynamic charts and dashboards.

5. Deployment

The application runs on a virtual machine and is available company-wide via VPN.
