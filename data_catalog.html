Data Catalog AI Agent

1. Problem Definition / Objective

This project focuses on building an AI-powered data catalog that enables company employees to discover and understand data assets using natural language queries. The solution bridges the gap between business users and complex data warehouses by allowing users to ask questions such as “Where can I find customer churn information?” and receive precise answers about the relevant tables and columns.

  • Create a centralized data catalog for thousands of database tables
  • Automatically generate clear and professional descriptions for tables and columns
  • Enable natural language search over data assets using an AI assistant
  • Reduce dependency on data teams for data discovery and interpretation

2. Data Available

  • Metadata containing table names, column names, and data types
  • More than +4k tables and +20k columns

3. Solution Architecture

The solution begins with automated metadata enrichment. Using the OpenAI API, table-level descriptions are generated by providing the model with each table’s name, its columns, and their data types. Column-level descriptions are then generated in a similar manner. These enriched descriptions form a structured knowledge base.

The resulting catalog is used as a Retrieval-Augmented Generation (RAG) source for a conversational AI assistant built on Google Vertex AI. When users submit natural language queries, the chatbot retrieves the most relevant table and column descriptions and generates accurate, contextual responses, guiding users directly to the data they need.

4. Tools and Technologies

  • Python
  • SQL
  • OpenAI API
  • Large Language Models (LLMs)
  • Google Vertex AI
  • Retrieval-Augmented Generation (RAG)

5. Business Impact

  • Faster data discovery across thousands of tables
  • Improved data accessibility for non-technical users
  • Reduced cognitive load and fewer interruptions to data teams
  • Increased adoption and trust in enterprise data assets
